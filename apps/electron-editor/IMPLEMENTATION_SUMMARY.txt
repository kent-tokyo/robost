ARCHITECTURE & DATA FLOW

Screenshot Capture Pipeline:
  
  RPA Server (HTTP)
       │
       ├─ Provides: GET /screenshot
       └─ Returns: PNG image data
              ↓
  Electron Main Process
       │
       ├─ IPC Handler: 'rpa:screenshot'
       ├─ Fetches PNG via http.get()
       ├─ Converts to base64
       └─ Returns to renderer
              ↓
  Renderer Process
       │
       ├─ useScreenCapture hook
       ├─ Manages fetch state
       ├─ Handles auto-refresh timing
       └─ Updates component state
              ↓
  ScreenPanel Component
       │
       ├─ Image canvas (display)
       ├─ Overlay canvas (regions)
       ├─ Coordinate tooltip
       └─ Sidebar sections
              ↓
  State Storage
       │
       ├─ useScreenCapture: Local screenshot
       ├─ useRunStore: Picked coordinates
       └─ ScreenPanel local: Zoom, pan, region

TROUBLESHOOTING REFERENCE

Issue: "RPA server not running"
Solution: Start scenario first, wait for port message

Issue: Screenshot appears blank
Solution: Check server logs, verify /screenshot endpoint exists

Issue: Coordinates not picking
Solution: Ensure screenshot is loaded, click on canvas, not outside

Issue: Zoom not working
Solution: Click screenshot first, then try zoom

Issue: Theme colors not correct
Solution: Check globals.css, verify CSS variables are set
